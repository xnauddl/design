시스템화 마법사

선택한 프레임을 한 번에 토큰화 · 바인딩 · 정돈 · 접근성 검수. 세부값은 ‘만들기/적용’ 탭의 각 입력(base·허용오차·맥락단계·대비 기준)을 사용합니다.
온보딩 / 가이드
처음이세요? ‘시스템화 마법사’로 추출 → 토큰 → 바인딩 → 정돈을 한 번에 끝낼 수 있어요. 단계별로 직접 하려면 ‘만들기·적용’ 탭을 차례로 사용하세요.

진행 안내

권장 순서와 단계 상태(전제 충족 여부). 클릭하면 해당 단계로 이동.
리네임 · 대비 점검 · 컴포넌트는 전제 없이 언제든 가능(독립).

0 · 브랜드 팔레트 생성

미리보기로 확인한 뒤 ‘적용’하면 변수로 반영됩니다.

1 · 추출

1.5 · 색 편집 (hue → 역할)

Global은 hue 패밀리(color/blue/500), 역할은 Semantic만. 역할을 확정해 ‘반영’하세요.

2 · 토큰 생성 (Global + Semantic)

미리보기로 변경 요약을 확인한 뒤 적용하세요.

변수 편집

Global·Semantic·Component 변수의 값·이름·스코프를 직접 고치거나 삭제합니다. (각 작업 단일 Undo)
즉시 편집 · 단일 Undo

2.5 · 시맨틱 매핑 (역할 → 토큰)

형식: 역할 = Global변수이름

2.6 · 텍스트 스타일 (화면 → 스타일)

선택한 텍스트에서 후보를 뽑아 텍스트 스타일로 등록하고 시맨틱 변수(font-size/line-height)에 바인딩합니다. (Pro)
이름크기행간스타일

3 · 적용 (바인딩)

진행률

4 · 리네임

되돌리기 안전장치
이 실행은 한 번의 되돌리기(Ctrl/⌘Z)로 전체를 취소할 수 있어요.

5 · 명도 대비 점검 (WCAG)

선택 안 텍스트 ↔ 배경 대비를 WCAG로 검사(읽기 전용).

컴포넌트 / 베리언트

하위에서 후보 스캔 → 골라 등록. 같은 베이스 이름(button/primary) → 베리언트 세트.

내보내기 (코드)

모든 디자인 변수(색·크기·간격·반경·타이포)를 코드로. HUG/FILL은 토큰이 아니라 제외.
